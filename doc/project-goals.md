# Project Corpus Forge - Team Goals & Overview

## What is Corpus Forge?

We're building a **web-based system** that helps users manage, organize, and interact with large collections of documents using AI.

### Key Terms
- **Corpus**: A large collection of organized text documents from various sources
- **Heterogeneous**: The documents can be different types and formats

---

## Mission

Design and implement a web-based system that transforms a heterogeneous corpus into actionable knowledge through:
- **Retrieval**: Search and find relevant documents intelligently
- **Prompting**: Ask AI questions and get insights from the document collection

### User Capabilities
Your system must enable users to:
1. **Import** documents into the system
2. **Organize** collections of documents
3. **Explore** documents through search and browsing
4. **Interact** with AI-powered workflows to extract knowledge

---

## Project Goals

The goal is **not** simply to build an application. Instead, we're demonstrating our ability to:

1. **Learn unfamiliar technical concepts quickly** — You'll encounter new frameworks, libraries, and tools
2. **Use AI-assisted development tools effectively** — Leverage AI to accelerate development
3. **Make sound engineering decisions under uncertainty** — Choose technologies and approaches with incomplete information
4. **Evaluate trade-offs** — Balance simplicity vs. robustness, speed vs. accuracy, scope vs. timeline
5. **Deliver a working software system** — The app must actually run and solve the core problem

---

## Discussion Topics for the Team

Before diving into code, discuss and align on these questions:

### Document Handling
- What types of documents should we support? (PDFs, text files, Word docs, web pages?)
- Should we have a size limit? (e.g., max 100MB per document)
- How should users upload documents? (Single file? Batch? URL?)

### Search & Retrieval
- What search features matter most? (Keyword search? Semantic search? Filters?)
- Should we support full-text search or just metadata?
- How do we handle large documents efficiently?

### AI Interaction
- How should users interact with the AI? (Chat interface? Pre-built prompts? Custom queries?)
- What AI service should we use? (OpenAI? Hugging Face? Local model?)
- What kinds of prompts should we support? (Summarization? Q&A? Analysis?)

### Technical Stack
- Frontend framework? (React? Vue? Vanilla JS?)
- Backend framework? (Flask? Django? Node.js?)
- Database? (PostgreSQL? MongoDB? SQLite?)
- Hosting? (Cloud? Local server?)

### MVP (Minimum Viable Product)
- What's the smallest useful version we can build?
- What features are essential vs. nice-to-have?
- What's our timeline and capacity?
